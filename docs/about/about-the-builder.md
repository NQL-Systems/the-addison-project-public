# About the Builder

I’m a self-taught builder working in AI systems, documentation, workflow design, and agent orchestration.

I do not come from a traditional software engineering background, and I’m still learning a lot of this as I go. I want to be honest about that, because I do not want to present myself as something I’m not. At the same time, I’m not treating this like casual experimenting either. I’m trying to build real things, understand them deeply, document them clearly, and improve my judgment as the work gets more complex.

A real reason I ended up exploring memory, artifacts, and persistent system structure is that I needed some of that support for myself too. I tend to generate a lot of ideas, directions, and project threads, and if I do not get them into a form I can return to and iterate on quickly, I can lose the thread or forget where I was trying to go. Part of this project came from wanting a system that could help hold state across work, preserve direction, and let me pick ideas back up without rebuilding everything from scratch every time.

A big part of how I work is through orchestration.

I use frontier models and coding agents to help me think, pressure-test ideas, draft, implement, compare approaches, audit work, and move faster than I could alone. But I do not see that as handing the work over. I see it as learning how to direct systems well, how to set constraints well, and how to keep truth and structure intact while using powerful tools.

**Code in this project may be drafted or implemented with agent assistance, but direction, constraints, review framing, and final project intent remain human-led.**

That distinction matters to me.

Part of what I’m exploring through this project is what becomes possible when someone outside a formal engineering path takes documentation seriously, learns in public, and uses agents with discipline instead of treating them like magic. I care a lot about keeping claims honest, separating what is proven from what is aspirational, and building in a way that leaves behind something other people can actually inspect.

I’m still learning. I expect that to stay true for a long time.

What I hope this repository shows is not that I have everything figured out, but that I’m building carefully, thinking seriously, and trying to make the process legible as I go. If someone with more experience sees gaps, clearer ways to frame something, better patterns, or mistakes worth correcting, that kind of feedback is valuable to me.

I’m not trying to posture as finished.

I’m trying to build something real, understand what I’m doing, and get better in public without pretending the road behind me was different than it was.
