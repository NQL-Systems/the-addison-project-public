# Addison

Addison is a local, daemon-backed state system built around persistent artifacts, deterministic context construction, and stateless inference.

This repository is a bounded public surface for the project. It contains public-safe documentation, proof artifacts, and inert examples. It does not contain live private memory, operator-specific identity, or the full private development authority.

## What Addison Is

- a personal state machine
- a local agent system
- artifact-backed continuity
- a daemon-authoritative core
- a human-governed system

## What Addison Is Not

- not a chatbot wrapper
- not a hidden-autonomy system
- not a multi-model round table
- not a production-ready platform

## Why It Exists

Most AI systems keep continuity inside the conversation or inside the model.

Addison is built in the opposite direction. It keeps memory, continuity, and system truth in persistent artifacts, uses deterministic context construction, and treats inference as a replaceable tool rather than as the system itself.

The goal is to explore a system where intelligence can be applied without surrendering control of state, boundaries, or operator authority.

## Public Proof

This repository is meant to show the public-safe shape of the system through a small set of bounded proof surfaces, including:

- artifact-gated startup behavior
- public-safe architecture documentation
- the distinction between Addison as the system and WBTSR as the workflow used to build it

## Method

Addison is the system. WBTSR is the workflow used to build it.

WBTSR stands for **Work, Build, Truth, Session, Rehydrate**. It is a human-governed method based on bounded work, explicit truth surfaces, separate builder and auditor roles, and restart-safe rehydration.

## Repository Boundaries

This repository is intentionally partial.

It may include:

- public-safe documentation
- proof artifacts
- inert templates and examples
- selected public-facing implementation/context surfaces

It does not include:

- live private memory
- operator-specific identity artifacts
- private prompts, logs, or transcripts
- runtime-local private state
- the full private development repo and its governing authority surfaces

Public material here should be read as a bounded representation of the project, not as the full internal system.

## Status

Active, experimental, and intentionally partial.
