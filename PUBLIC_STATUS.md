## 2. Current Public Posture

Addison is currently presented here as:

- a local, daemon-backed state system
- built around persistent artifacts
- using deterministic context construction
- treating inference as a stateless tool
- operating under explicit human-governed boundaries

This repository is intentionally partial.

It is meant to expose the public-safe shape of the system, not the full private runtime, memory, or development authority.

---

## 3. What Is Currently Proven

The public surface currently supports these claims:

### 3.1 Architectural Shape

The core system architecture is real and deliberate:

- the daemon is the authority layer
- persistent artifacts hold continuity and system truth
- context construction is deterministic
- inference is downstream of prepared context
- runtime surfaces are consumers rather than truth owners

### 3.2 System / Method Distinction

The public surface preserves the distinction between:

- **Addison** — the system
- **WBTSR** — the workflow used to build it

These are related, but they are not the same thing.

### 3.3 Public Boundary Discipline

The project follows a bounded public/private posture:

- public surfaces are intentional
- live private memory is not public
- operator-specific identity is not public
- templates/examples are inert and not live runtime truth

### 3.4 Experimental Reality

Addison is a real system under active development, but it is still experimental.

This repository does not present it as complete or production-ready.
